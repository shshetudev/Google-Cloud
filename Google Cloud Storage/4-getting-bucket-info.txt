Here we can get information on size and metadata of our Cloud Storage buckets.

Key terms:
-----------
1) Storage Classes: 
===================
	i) The storage class we set for an object affects the object's avilability and pricing model. 
	ii) We can change the storage class of an existing object either by rewriting the object or by using Object Lifecycle management.
	iii) 
	Storage Class   -> for APIs, gsutil -> Min Storage duration
	Standard Storage -> STANDARD -> None
	Nearline Storage -> NEARLINE -> 30 days
	Coldine  Storage -> COLDLINE -> 90 days
	Archive  Storage -> ARCHIVE  -> 365 days



Determining a bucket's size:
============================
	`gsutil du -s gs://demo-bucket-shetu`

a) The `gsutil du` command calculates space usage by making object listing requests, which can take a long time for large buckets.
b) If the number of objects in our bucket is hundreds of thousands or more, we should use `Monitoring` instead.

Displaying a bucket's metadata:
===============================

gsutil:
-------
	`gsutil ls -L -b gs://demo-bucket-shetu`

-------------
